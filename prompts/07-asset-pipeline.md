# 素材自动化流水线 Prompt

```text
请检查当前《沫沫的珊瑚午餐》，专项建立或修复可复现的素材处理、验证和报告流水线。不要只写文档；实现可运行脚本，处理现有原图，并让运行时资产和报告由脚本产出。

目录约定：workflow/sources/ 保存角色 3×2 原图、可选的角色循环视频和 4×4 UI atlas；workflow/intermediate/ 保存抽帧结果、本地或外部抠图 mask、处理前后中间文件与 provenance；workflow/tools/ 保存处理与构建脚本；assets/images/ 保存运行时透明素材；workflow/reports/ 保存验证 JSON、manifest 和 contact sheet。

实现背景连通的 #ff00ff 容差色键透明化、边缘去品红/紫色溢色、分格、连通域清理、主体裁切、最近邻缩放、锚点注册、横向 spritesheet 合成。不能只匹配精确 RGB；使用颜色距离或 HSV 容差识别生图、压缩和抗锯齿产生的近似紫色，再结合从画布四边 flood-fill 的背景 mask 与透明边缘去色，避免全局删除角色本身的紫色/粉色。保留主体附近合理的小部件，但清除跨格碎片和邻格粘连。鱼类按头部或身体中心对齐，水母按钟体顶部；不要手写大量逐帧偏移掩盖错误。

建立背景移除的分级 fallback：①重新生成更干净、颜色更统一的背景；②改进本地容差色键、背景 flood-fill、alpha mask 和 edge despill；③若脚本仍无法正确处理复杂鳍、触须、半透明钟体或紫色污染，使用可免费访问的外部 AI remove-background 工具、图像编辑服务或环境内置抠图能力。外部工具只处理确认可外发、无敏感信息且许可允许的素材；保存处理前后文件和 mask，并记录工具名称、URL、使用日期、参数/操作、条款或许可。下载后的透明文件必须完全本地化，外部服务不得成为运行时或可复现构建的硬依赖。

外部 AI 输出继续进入同一验证链，并与原图做 alpha 叠加或差分检查：确认鱼鳍、鱼尾、触须、触手、头灯、背包、半透明高光和内部孔洞没有被误删或填死。必要时只采用 AI 生成的 mask 与原图重新合成，不能因为工具声称“背景已移除”就直接通过。

增加分辨率预算步骤：从 manifest 中读取每个物种的最大运行时绘制尺寸、最大成长倍率和目标 DPR，反推最低单帧尺寸。普通鱼、玩家和鲨鱼不必使用相同像素预算；鲨鱼王等大型主体生成更大的源图/运行时帧或高分辨率变体。禁止依靠浏览器把细节不足的低分辨率帧持续放大；同时确保 Canvas `imageSmoothingEnabled = false`，DOM 图片使用适合像素风的 `image-rendering`，并尽量采用稳定整数缩放。

流水线同时支持两种动画源。当前环境没有视频生成能力时，处理 3×2 ImageGen sprite sheet；未来若有可靠视频生成能力，则支持读取固定镜头、固定朝向、纯色背景的短循环视频，用 ffmpeg 等工具定距抽帧，再执行背景移除、统一画布、像素化、锚点对齐和 spritesheet 合成。视频路线必须额外检测循环首尾差异、主体/装备形体漂移、镜头位移、运动模糊和帧间身份变化；它通常能提供比独立图片帧更连贯复杂的动作，但不能绕过验证。

验证必须检查：输入格数和尺寸、透明四角、精确及近似色键残留、半透明 alpha 边缘色溢、主体触边、覆盖率、跨格碎片、输出尺寸、帧数、锚点误差、最大放大倍率和有效分辨率。动画目标 8 张 spritesheet/48 帧、最大锚点误差 ≤0.5 px；各帧保持一致画布、显示尺寸和稳定视觉中心，不能按透明包围盒分别缩放。报告记录源尺寸、单帧尺寸、最大运行时绘制尺寸、目标 DPR、放大倍率、残留像素数量/比例；超出清晰度预算或色键残留阈值时脚本返回非零。UI 目标为 4×4 atlas 的 16 个基础素材，加 1 个独立成长图标，共 17 个透明素材；要求无色键残留和裁切触边，并核对每个运行时 CSS/JS 资源键都有实际文件映射。对九宫格或分片框体额外验证 inset、固定角尺寸、单轴边条、中心安全区、拼接接缝及 normal/hover/active 几何一致性，并输出可供运行时读取或文档核对的元数据。失败信息必须指出具体文件、帧、残留颜色范围或分辨率不足项。

生成四张带名称、清晰底色和统一间距的报告图：animation-contact-sheet.png、scene-contact-sheet.png、depth-contact-sheet.png、ui-contact-sheet.png。动画表显示 8×6 帧；UI 表按 4 列展示 16 个 atlas 切片和 1 个独立成长图标；场景和景深显示实际运行时素材。把 JSON 和四张图打包进发布目录，并确保 README 相对链接可用。

脚本使用跨平台相对路径，不写死用户主目录；重复运行结果稳定，不破坏手工文档。执行全部脚本，核对文件数量、尺寸和 hash/内容，检查报告与运行时实际资产一致。自动指标之后仍要做两类肉眼复核：在深蓝、白色和高饱和对比底上查看透明边缘；在游戏内以最大显示尺寸和高 DPR 查看鲨鱼等大型素材。简短记录命令和结果。
```
