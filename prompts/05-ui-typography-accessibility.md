# UI、字体与无障碍 Prompt

```text
请检查当前《沫沫的珊瑚午餐》，专项完善 HUD、封面/结算界面、响应式布局、字体和无障碍。直接修改并在多尺寸浏览器中验证；不得改变游戏规则。

HUD 包含得分、成长条与百分比、当前关卡/3、海域名、体型阶段、三颗泡泡心、暂停、观赏模式、声音和音量。封面包含英文小副标题、中文标题、玩法说明、吃/躲规则、手动开始与水族馆观赏入口。游戏中包含 toast、底部操作提示、下一关按钮；结算和失败页显示总分与重开。

UI 采用 Glass + Water 混合方案：正常文档流或 flex/grid、圆润 border-radius、半透明青蓝渐变、内高光和克制阴影负责动态容器；与角色、场景统一的独立透明像素图标、小型角饰和少量固定装饰负责游戏感。`backdrop-filter` 仅作增强，并提供不依赖模糊的清晰后备。避免方正切角、硬质科幻八边形、复杂三栏或每块都有粗边框的面板；对儿童要直观、轻盈。

主对话框等少数固定装饰可以使用九宫格、分片 DOM 或 `border-image`，但必须明确 inset：四角固定尺寸，四条边只能沿单轴平铺/拉伸，中心区域可扩展且有足够文字安全区。CSS 仍负责文字、对齐、换行和动态宽度；贴图不能烘焙文字，也不能直接缩放整张 frame。按钮的 normal/hover/active 状态共用一致几何结构。若出现拉伸、黑边、文字错位、padding 失衡、hover 变形或边角接缝，优先减少 frame 的覆盖范围，让动态主体回到 Glass + Water，而不是用更多覆盖规则掩盖问题。

主按钮、图标按钮、成长条、生命和音量都保持圆润。冲刺提示使用紧凑的鳍、泡泡或圆形反馈，不要做成突兀的长矩形。hover/active 只做轻微亮度、水光、浮起或压下，不得切角、横向拉伸、错位、露出错误底色。点击区域至少 44×44 px。成长图标放在标签或进度条左侧等自然位置，不能悬在底部；成长、冲刺、观赏等相邻功能不要重复使用同一图标。所有图标必须完成 CSS/JS 资源映射并在运行时实际显示；成长条、音量滑杆等依赖渐变或伪元素的填充也要在浏览器确认可见。

字体禁止联网加载。使用 "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Noto Sans CJK SC", system-ui, sans-serif，并设置 font-synthesis: none。不要把 Avenir 放在中文字体之前。中文、数字和英文必须协调：HUD 数字、百分比、Shift 等中英混排和封面英文使用约 600–700 权重，禁止 900/950 让西文显得过黑；数字启用 tabular-nums。检查所有动态文本，不只检查静态封面。

响应式覆盖桌面、手机竖屏和低高度横屏；处理 safe-area。HUD 不重叠、不越界，封面不裁切，长海域名可缩短但信息不丢失。逐块检查文字与图标垂直居中、基线、动态宽度、上下 padding、透明度和核心面板留白，不能出现内容整体偏上、底部空一截或图标占位却未显示。提供 aria-label、aria-live、role=meter、aria-valuemin/max/now、aria-pressed；键盘焦点清楚，纯图标按钮有可读名称。prefers-reduced-motion 下关闭非必要循环动效。

UI atlas 保持 4×4 共 16 项：HUD、成长框、生命框、圆按钮、对话框、toast、tip、行动按钮、star、heart、pause、music、play、boost、pearl、warning；另生成独立的成长图标（金色扇贝、青色上升箭头与少量气泡），总计验证 17 项。所有素材无文字、无粘连。若框体需要九宫格，应额外保存准确 inset/分片元数据和可复现处理脚本；不能为了“用上素材”强行接入没有通过响应式验收的 frame。生成 ui-validation.json 和 ui-contact-sheet.png。完成后在真实浏览器检查 1280×720、390×844、844×390 和至少一个中间缩放尺寸；读取计算字体确认英文/数字权重，截图核对边角、接缝、拉伸、垂直对齐、padding、图标、动态宽度和 hover，不要只凭 CSS 源码判断，并检查控制台无错。
```
